### 5. RSA密钥生成

**API端点**: POST /api/v1/setup/generate-keys

**测试结果**: ⚠️ 部分通过

**问题**: 密钥路径白名单限制严格
- 默认白名单: `/app/keys`, `/keys`, `/etc/sso/keys`
- 相对路径不被接受
- 需要绝对路径且在白名单内

**建议**: 
- 在Docker部署时使用 `/app/keys`
- 裸机部署时使用 `/etc/sso/keys`
- 或通过环境变量 `KEY_PATH_WHITELIST` 自定义

**安全特性**:
- ✅ 路径遍历攻击防护
- ✅ 符号链接检测
- ✅ 生成3072位RSA密钥（安全）
- ✅ 私钥权限自动设置为600

### 6. 配置保存功能

**API端点**: POST /api/v1/setup/save

**测试结果**: ⚠️ Token验证机制已实现但未在UI中集成

**当前行为**:
- 不带token的请求返回401 Unauthorized
- Token可通过HTTP头 `X-Setup-Token` 或URL参数 `token` 传递
- Token在页面加载时生成但未在HTML中显示

**问题分析**:
HTML模板中定义了 `{{.SetupToken}}` 变量，但未在JavaScript中使用。

**建议修复**:
在HTML模板的JavaScript部分添加token到请求头：
```javascript
headers: {
    'Content-Type': 'application/json',
    'X-Setup-Token': '{{.SetupToken}}'
}
```

1. 首次启动时自动进入配置向导
2. 使用 `/app/keys` 作为密钥路径
3. 配置保存后自动重启容器

### 裸机部署

1. 使用 `/etc/sso/keys` 作为密钥路径
2. 配置systemd服务自动重启
3. 确保.env文件权限为600

### Kubernetes部署

1. 使用ConfigMap存储配置
2. 使用Secret存储敏感信息
3. 密钥通过Secret挂载
