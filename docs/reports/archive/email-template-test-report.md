### 2. 邮件发送测试 - **失败** ❌

#### 错误信息
```
526 Authentication failure[0]
```

#### 失败原因分析

**SMTP配置：**
- 服务器：`smtp.qiye.aliyun.com`
- 端口：`465` (SSL) / `587` (STARTTLS)
- 用户名：`system@eninte.com`
- 发件人：`system@eninte.com`

**可能的原因：**

1. **SMTP密码过期** ⚠️
   - `.env.test` 中的密码可能已失效
   - 阿里云企业邮箱密码可能已更改

2. **需要授权码** 🔑
   - 某些邮件服务商（如阿里云企业邮箱）要求使用"授权码"而非账户密码
   - 需要在邮箱管理后台生成专用的SMTP授权码

3. **IP白名单限制** 🌐
   - 阿里云企业邮箱可能限制了发送IP地址
   - 当前服务器IP可能未在白名单中

4. **账户安全策略** 🔒
   - 账户可能启用了二次验证
   - 需要在邮箱设置中允许"不够安全的应用"访问

#### 建议解决方案

1. **更新SMTP凭据**
   ```bash
   # 登录阿里云企业邮箱管理后台
   # 生成新的SMTP授权码
   # 更新 .env.test 中的 SMTP_PASSWORD
   ```

2. **验证SMTP配置**
   ```bash
   # 使用telnet测试SMTP连接
   telnet smtp.qiye.aliyun.com 465
   
   # 或使用openssl测试SSL连接
   openssl s_client -connect smtp.qiye.aliyun.com:465
   ```

3. **检查IP白名单**
   - 在阿里云企业邮箱后台添加当前服务器IP到白名单

4. **使用其他SMTP服务**
   - 考虑使用SendGrid、Mailgun等第三方邮件服务
   - 或使用个人邮箱（Gmail、QQ邮箱）进行测试

## 🔧 测试工具

### 1. 邮件发送测试脚本

**文件：** `scripts/test_email.go`

**用法：**
```bash
go run scripts/test_email.go \
  -to rhdnhc@qq.com \
  -type verification \
  -username "测试用户"
```

**参数：**
- `-to`: 收件人邮箱（必填）
- `-type`: 邮件类型（`verification` 或 `password_reset`）
- `-username`: 用户名
- `-env`: 环境配置文件路径（默认 `.env.test`）

### 2. 模板渲染测试脚本

**文件：** `scripts/render_email_template.go`

**用法：**
```bash
go run scripts/render_email_template.go \
  -type verification \
  -lang zh \
  -username "测试用户" \
  -output /tmp/email.html
```

**参数：**
- `-type`: 邮件类型（`verification` 或 `password_reset`）
- `-lang`: 语言（`zh` 或 `en`）
- `-username`: 用户名
- `-output`: 输出HTML文件路径（可选）
