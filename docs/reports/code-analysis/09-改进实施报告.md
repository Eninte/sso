### 8.3 修复: 指标调用问题

**问题**: 虽然定义了很多指标，但业务代码中没有实际调用 `Increment()` 或 `Set()` 方法。

**初始方案**: 使用全局变量 `globalMetrics` + `SetMetrics()` 函数。

**改进方案**: 采用依赖注入模式，通过构造函数传入 metrics 服务，消除全局状态。

**解决方案**:

**1. 添加HTTP中间件** (`cmd/server/main.go`)
```go
// 12. 应用中间件
router.Use(middleware.SecurityHeaders)
router.Use(middleware.Logger)
router.Use(metricsSvc.HTTPMiddleware)  // 自动记录HTTP请求指标
```

**2. AuthService 接受 metrics 依赖注入** (`service/auth.go`)
```go
type AuthService struct {
    store           store.Store
    passwordSvc     *crypto.PasswordService
    jwtSvc          *crypto.JWTService
    maxAttempts     int
    lockoutDuration time.Duration
    metricsSvc      *metrics.MetricsService // 可选，通过构造函数注入
}

func NewAuthService(
    store store.Store,
    passwordSvc *crypto.PasswordService,
    jwtSvc *crypto.JWTService,
    maxAttempts int,
    lockoutDuration time.Duration,
    metricsSvc ...*metrics.MetricsService, // 可变参数，向后兼容
) *AuthService {
    var m *metrics.MetricsService
    if len(metricsSvc) > 0 {
        m = metricsSvc[0]
    }
    return &AuthService{
        store:           store,
        passwordSvc:     passwordSvc,
        jwtSvc:          jwtSvc,
        maxAttempts:     maxAttempts,
        lockoutDuration: lockoutDuration,
        metricsSvc:      m,
    }
}

func (s *AuthService) incrementMetric(name string) {
    if s.metricsSvc != nil {
        s.metricsSvc.Increment(name)
    }
}
```

**3. 在业务方法中调用指标** (`service/auth.go`)
```go
// Register 成功时
s.incrementMetric("auth_register_total")

// Login 成功时
s.incrementMetric("auth_login_total")

// Login 失败时
s.incrementMetric("auth_login_failed_total")

// 账户锁定时
s.incrementMetric("auth_account_locked_total")

// RefreshToken 成功时
s.incrementMetric("auth_token_refresh_total")

// Logout 时
s.incrementMetric("auth_token_revoke_total")
```

**4. 在main.go中通过构造函数传入metrics**
```go
// 7. 初始化指标服务
metricsSvc := metrics.NewMetricsService()

// 8. 初始化业务服务（通过构造函数注入 metrics）
authSvc := service.NewAuthService(
    store,
    passwordSvc,
    jwtSvc,
    cfg.MaxLoginAttempts,
    cfg.LockoutDuration,
    metricsSvc,
)
```

**改进对比**:

| 项目 | 旧方案 (全局变量) | 新方案 (依赖注入) |
|------|------------------|------------------|
| 全局状态 | ✅ 有 `globalMetrics` | ❌ 无全局状态 |
| 可测试性 | 需要调用 `SetMetrics(nil)` | 直接传 nil 即可 |
| 并发安全 | `SetMetrics` 非线程安全 | 构造时一次性注入 |
| 向后兼容 | - | 可变参数 `metricsSvc ...` |

**效果**: 
- 所有HTTP请求都会自动记录指标
- 业务事件（登录、注册、Token操作）都会记录指标
- `/metrics` 端点返回有意义的数据
- 消除全局状态，提升可测试性和并发安全性

**语言中间件** (`middleware/language.go`):
```go
func Language(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        lang := detectLanguage(r) // 检测语言
        ctx := context.WithValue(r.Context(), LanguageKey, lang)
        next.ServeHTTP(w, r.WithContext(ctx))
    })
}
```

```go

// 英文响应 (通过 ?lang=en-US)

```
