## 🔒 安全最佳实践

### 开发时
1. ✅ 使用参数化查询（防止SQL注入）
2. ✅ 验证所有用户输入
3. ✅ 使用`crypto/rand`生成随机数
4. ✅ 敏感操作记录审计日志
5. ✅ 错误消息不暴露内部信息

### 部署时
1. ✅ 设置强HMAC密钥（32字节）
2. ✅ 生产环境bcrypt cost >= 12
3. ✅ 启用数据库SSL
4. ✅ 配置正确的CORS来源
5. ✅ 定期备份数据库

### 运维时
1. ✅ 监控登录失败率
2. ✅ 监控限流触发次数
3. ✅ 定期检查审计日志
4. ✅ 定期更新依赖库
5. ✅ 定期进行安全审计
