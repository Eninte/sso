# SSO服务 - 压力测试现场清单

本文档用于现场执行压力测试时逐项勾选，配合 [PRESSURE_TESTING_RUNBOOK.md](./PRESSURE_TESTING_RUNBOOK.md) 一起使用。

## 1. 使用方式

- 本文档面向实际执行人，强调“先检查，再执行，再记录”
- 勾选顺序建议：环境 → 数据 → 工具 → 场景 → 结果 → 复盘
- 每次压测都复制一份结果记录模板，避免覆盖历史记录
- 若本次目标是容量测试，不要混入保护模式结论
- 若本次目标是保护测试，不要用吞吐结果作为容量基线

## 8. 结果判定清单

### 8.1 容量场景

- [ ] 5xx 小于既定门槛
- [ ] P95 没有持续恶化
- [ ] 稳定吞吐已明确
- [ ] 拐点已明确
- [ ] 瓶颈位置已判断

### 8.2 保护场景

- [ ] 429 行为符合预期
- [ ] 401 行为符合预期
- [ ] 409 行为符合预期
- [ ] 账户锁定符合预期
- [ ] 保护生效时系统未雪崩

### 8.3 稳态场景

- [ ] 内存进入平台期
- [ ] goroutine 进入平台期
- [ ] 连接池进入平台期
- [ ] 没有持续性错误率爬升
- [ ] 峰值结束后可恢复

## 10. 常见误区检查

- [ ] 没有把保护模式结果当作容量结论
- [ ] 没有把 409 冲突当作系统故障
- [ ] 没有把 OAuth 非法请求 4xx 当作系统异常
- [ ] 没有把场景内临时登录混入 userinfo 压测
- [ ] 没有忽略 `BCRYPT_COST`
- [ ] 没有忽略限流配置
- [ ] 没有遗漏依赖服务状态记录
