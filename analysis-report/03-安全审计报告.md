## 五、修复建议

### 5.1 紧急修复 (P0)

#### 1. 修复 `generateRandomString`
```go
// internal/service/oauth.go
import "crypto/rand"
import "encoding/base64"

func generateRandomString(length int) (string, error) {
    bytes := make([]byte, length)
    if _, err := rand.Read(bytes); err != nil {
        return "", fmt.Errorf("生成随机数失败: %w", err)
    }
    return base64.RawURLEncoding.EncodeToString(bytes)[:length], nil
}
```

#### 2. 修复 MFA 密钥生成
```go
// internal/service/mfa.go
func generateTOTPSecret() (string, error) {
    secret := make([]byte, 20) // 160位，符合RFC 4226
    if _, err := rand.Read(secret); err != nil {
        return "", fmt.Errorf("生成TOTP密钥失败: %w", err)
    }
    return base32.StdEncoding.EncodeToString(secret), nil
}
```

#### 3. 添加密钥路径验证
```go
// internal/crypto/keyloader.go
import "path/filepath"

func validateKeyPath(path string) error {
    cleanPath := filepath.Clean(path)
    if strings.Contains(cleanPath, "..") {
        return errors.New("密钥路径包含非法字符")
    }
    
    info, err := os.Stat(cleanPath)
    if err != nil {
        return err
    }
    
    // 检查私钥文件权限
    if info.Mode().Perm()&0077 != 0 {
        return errors.New("密钥文件权限过于宽松")
    }
    
    return nil
}
```

#### 4. 修复管理员权限检查
```go
// internal/handler/admin.go
func (h *AdminHandler) isAdmin(r *http.Request) bool {
    userID := middleware.GetUserIDFromContext(r.Context())
    user, err := h.store.GetByID(r.Context(), userID)
    if err != nil {
        return false
    }
    // 使用专门的角色字段，而非邮箱域名
    return user.Role == "admin"
}
```

## 七、结论

该项目存在多个**严重安全漏洞**，核心安全组件（crypto、oauth、mfa）的实现存在致命缺陷。主要风险包括：

1. **Token可预测** - 使用非加密安全的随机源
2. **MFA可绕过** - TOTP密钥生成不安全
3. **权限提升** - 管理员检查可绕过
4. **文件泄露** - 密钥路径未验证
5. **已知漏洞** - 22个未修复漏洞

**强烈建议**: 在修复所有 P0 级别漏洞之前，**禁止部署到生产环境**。
